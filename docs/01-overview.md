# 01. 개요

이 문서는 `SwarmUI`를 실제 작업 환경에서 어떻게 배치하고 어떤 사람이 무엇을 책임지는지 설명한다.  
핵심은 간단하다.

- 작업자 PC는 가볍게 유지한다.
- GPU 서버가 실제 연산을 담당한다.
- `SwarmUI`는 작업자와 서버를 연결하는 UI다.
- `StabilityMatrix`는 서버 측 패키지와 모델을 정리한다.

즉, 이 매뉴얼은 “어떻게 예쁜 그림을 만드는가”가 아니라 “작업자가 일을 하고, 서버가 계산을 대신하는 구조를 어떻게 만든다”를 설명한다.

## 역할 분리

- `작업자`
  - `Generate`와 `Simple`을 주로 쓴다.
  - 결과 확인, 재시도, 프롬프트 수정, 큐 관리에 집중한다.
  - 워크플로 그래프를 직접 만들 책임은 없다.
- `워크플로 작성자`
  - `Comfy Workflow`에서 원본 그래프를 만들거나 수정한다.
  - `Simple`에 노출할 입력값을 정리한다.
  - 작업자용 템플릿을 준비한다.
- `운영자`
  - `Server`, `Utilities`, `User`, `Backends`, `Logs`, `Extensions`를 관리한다.
  - 모델 경로, 권한, 원격 접속, 백엔드 병렬화를 책임진다.

## 탭 한눈에 보기

| 탭 | 주 사용 대상 | 역할 |
|---|---|---|
| `Simple` | 작업자, 템플릿 소비자 | 승인된 워크플로를 단순한 입력 폼으로 제공 |
| `Generate` | 대부분의 작업자 | 기본 생성 화면, 모델 선택, 프롬프트, 큐, 히스토리, Grid |
| `Comfy Workflow` | 워크플로 작성자, 파워 유저 | 노드 그래프를 직접 다루는 고급 작업 |
| `Utilities` | 운영자, 파워 유저 | 모델 다운로드, 메타데이터 재검사, 변환 도구 |
| `User` | 모든 계정 | 개인 설정, 출력 경로, 비밀번호, 계정 관련 기능 |
| `Server` | 운영자 | 백엔드, 사용자 관리, 로그, 확장, 서버 설정 |

## SwarmUI를 쓰는 이유

SwarmUI는 ComfyUI를 대체하는 것이 아니라, ComfyUI 위에 작업자 친화적인 UI와 운영 도구를 얹는다.  
공식 문서가 강조하는 장점은 다음과 같다.

- 워크플로 브라우저와 공유가 편하다.
- Generate 탭에서 일상 작업을 빠르게 처리할 수 있다.
- Grid Generator로 조건 비교를 쉽게 돌릴 수 있다.
- Simple 탭으로 비전문가용 인터페이스를 만들 수 있다.
- API와 여러 백엔드를 함께 써서 자동화와 병렬 처리를 할 수 있다.
